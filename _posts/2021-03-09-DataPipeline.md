### 데이터 파이프라인?

데이터 파이프라인에 앞서 ETL에 대해서 알고 지나가자. 

**ETL**은 추출(**E**xtract), 변환(**T**ransform), 적재(**L**oad) 의 줄임말이다. ETL 시스템은 하나의 시스템에서 데이터를 추출하고 변환하여 데이터베이스에 저장한다. ETL은 보통 배치로 작동하며 대용량의 데이터를 특정 시간에 한 공간에 저장한다. (배치 : 일단 특정 시간에 작동하게 한다고 알고 있자.)

**데이터 파이프라인**은 이 ETL을 포함하고 있는 말이다. 일단 추출하고 변환하고 적재를 위한 것은 맞다. 그러면 무엇이 다를까? 데이터 파이프라인은 여러 수동적인 작업을 제거하고 각 데이터가 지속적으로 추출,변환,적재 과정을 수행하게 하는 시스템을 말한다. 물이 흘러가는 것(Flow)처럼 자동화 하는 것이 포인트다.

쉽게 표현하자면, 여기저기서 끌어모은 데이터들을 적절히 처리해서 데이터 허브에 저장하는 데이터 통로다. 안내 표지판이 다 있어서 어디서 어떻게 어디로 갈지 이미 다 정해져 있는 것이다.

### 그래서?

한 가지 예시를 생각해보자. 우리는 영화에 대한 데이터를 분석을 하려고 한다. 여기저기서 영화에 대한 데이터들을 끌어모아서 그걸 가공을 해야하는데 어떤 기준으로 분류를 할까.

        감독? 배우? 개봉 날짜? 장르? 

일단 영화 제목 안에서 해당 영화와 관련된 정보들을 모아야 한다. 승리호로 예를 들어보자. 

2021.김태리.송중기.SF.줄거리. 등등

승리호에 관련된 정보들을 나열했지만 이보다 더 많은 정보들이 있을 것이다. 그 데이터들 중에서 어떤 데이터를 어떠한 형태로 가공해서 분석에 사용할지는 데이터 분석을 하는 사람이 정할 것이다. 그러면 일단 그 정보들을 모두 다 저장을 해야 한다. 어떤 데이터가 쓰일지 모르니까. 

그러면 우리는 첫번째로 해야할 것이 여기저기서 모은 승리호에 관련된 정보들을 모두 하나의 틀 안에 집어 넣어야 한다.  그리고 그 틀이 보통 CSV 혹은 JSON 파일로 형성이 된다. 

### 데이터 수집

그러면 데이터 수집은 어떻게 할까? 

- Stream Processing : 스트리밍을 통해 얻은 데이터를 실시간으로 처리한다.

    - 보통 시계열 데이터 베이스에 저장이 된다.
    - 장기적으로 쓰기에는 무리가 있다.
    - Kafka가 여기에 해당한다고 볼 수 있다.
- Batch Processing : 실시간이 아닌, 어느 정도 정리된 데이터를 효율적으로 가공해서 처리한다.

그렇게 수집된 데이터들은 Database 나 Data-warehouse 에 저장이 된다.
